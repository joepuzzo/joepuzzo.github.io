User Threads

  • handled by threads library
  • threads share the parents process time on the cpu
  • cheaper then kernal

Kernal Threads

  • handled by the OS
  • OS schedules threads on CPU
  • multiprocess on system

Threads Model

  • many to one model: multiple user threads on a single kernal thread
  • one to one model: a single user thread is on a single kernal thread
  • many to many model: user threads are run on many kernal threads

LWP: Light Weight Process

  • A thread can read/write another threads vars
  • No protection between threads
  • Design threads carefully

Advantages of threads over processes

  • Threads share address space - very useful for several applications
  • less system overhead to
  • scheduling your kernal threads on multiprocessor systems allows your process to get more CPU time concurrency

Signals

  • notify a process that an event has occurred
